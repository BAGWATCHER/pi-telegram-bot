# Gauntlet Mirror Training Loop

## Daily loop (2-4 hours)

1. 15m — Plan update
   - confirm today’s objective
   - define done criteria
2. 60-120m — Build sprint
   - implement one narrow vertical slice
3. 20m — QA/eval run
   - run tests/evals
   - capture failure patterns
4. 20m — Debrief note
   - add 1 source-backed insight
   - add 1 drill for next session

## Weekly loop

- Mon: planning + architecture decisions
- Tue/Wed: implementation
- Thu: hardening (evals, observability, rollback/retry)
- Fri: demo + postmortem + teach-back artifact

## Deliverables per week

- One deployable artifact
- One test/eval report
- One teachable lesson for agents/people
- One updated context bundle

## Quality bar

- Reproducible setup
- Clear source attribution
- Objective evaluation criteria
- Explicit tradeoffs and risk notes
